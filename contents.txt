Image embedding features
1) colour histogram-colour features (produces features using the number in a particular bin)
2) Local Binary pattern- used for features(produces features using histogram bins)
3) gabor filters - texture detection (number of features is the summary statistic for different hyper parameters)
4) HOG count of gradients pointing in the same direction
5) SIFT / SURF / ORB / BRIEF
6) Discrete Cosine Transform (DCT) coefficients
7) Wavelet transforms (e.g., Haar, Daubechies)
8) Saliency map statistics
9) Objectness / EdgeBoxes / region proposals



Why per-block (cell) features are better
Locality matters:
Animals may only occupy a part of the image (some grid cells).
Applying features to the whole image blurs local cues — edges, texture, color histograms will mix background + animal, making classification harder.
Grid-level labeling requirement:
Your project requires a label for each cell (0/1).
Extracting features per cell aligns naturally with the final grid-level classification.
Noise reduction:
Small sub-blocks allow local clustering of features.
Majority-vote or clustering within the block can reduce misclassification from outlier pixels.
